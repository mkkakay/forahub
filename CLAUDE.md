# ForaHub Claude Code Standards

## Before marking ANY task complete, verify all of the following:

### Technical
- Build passes with zero errors and zero warnings
- No console errors or hydration issues
- No broken images, logos, or assets
- No failed network requests for static assets

### Visual QA
- Spacing, sizing, and typography feel intentional not default
- Components match ForaHub visual identity: dark navy, clean white, Inter font
- Nothing looks placeholder, generic, or unfinished
- Premium institutional feel: think WHO, WEF, World Bank, Bloomberg Live

### Content and Imagery
- Images match the actual topic, event, or organization
- Logos belong to the correct organization
- Global South representation visible, not just Western institutions
- No misleading or irrelevant imagery

### Interaction
- Hover states work correctly
- Animations are smooth and not distracting
- Mobile layout is clean and usable
- Sticky and floating elements do not obstruct content

### Self-critique before finishing
Ask: Would this ship on a real UN or WEF platform?
Ask: Would a WHO professional trust this immediately?
Ask: Does anything still look broken, blurry, or unfinished?

If the answer to any of these is no, continue refining before reporting done.

### Final response format
Every completed task must include:
1. What was changed
2. What was visually verified
3. Any additional fixes made after review
4. Any remaining concerns or known limitations

A passing build alone does not mean a task is complete.
